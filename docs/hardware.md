## 확정된 하드웨어 설계

| 항목 | 확정 내용 |
|------|----------|
| 기본 키트 | KAIST 캔위성 기본 키트 (ATmega128) 그대로 사용 |
| 추가 컴퓨터 | RPi Zero 2W를 추가 탑재체로 연결 (ATmega128 교체 아님) |
| 카메라 | 기본 카메라 제거 → Global Shutter Camera 2개로 교체 |
| 짐벌 | MPU6050 + Servo 액티브 틸트 (항상 지면 방향 60~70° 유지) |
| 이미지 저장 | 고화질 사진 전부 SD카드에 로컬 저장 |
| 네트워크 통신 | 낙하 중 Bluetooth 텔레메트리만 (이미지 무선 전송 없음) |

## 하드웨어 연결 주의사항

### ATmega128 ↔ RPi Zero 2W 시리얼 연결

```
ATmega128 (5V) ↔ RPi Zero 2W (3.3V)
  → 레벨 컨버터(Level Shifter) 필수. 직결 절대 금지.
  → 5V를 3.3V GPIO에 직결하면 RPi가 손상됨
```

### Bluetooth 초기 설정

```
공장 기본값 = 9600bps
→ 지상 점검 모듈 또는 ATmega128 초기화 루틴으로 115200bps 설정 필요
→ 설정 안 하면 통신 속도 부족으로 텔레메트리 누락 발생
```

### GPS 파싱

```
$GPGGA, $GNGGA 두 형식 모두 처리 필요
위도: 3622.3345  →  36 + (22.3345 / 60) = 36.3722°
경도: 12721.9692 → 127 + (21.9692 / 60) = 127.3661°
```

```

  PHASE A: 낙하 중 — 캔위성 상공 비행
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  RPi Zero 2W
    ├─ Global Shutter Cam × 2  →  0.5s 간격 촬영
    ├─ MPU6050 + Servo          →  짐벌 자세 보정 (지면 방향 유지)

  노트북 (지상국)
    └─ Bluetooth 수신  →  텔레메트리 + Blur 수치 실시간 모니터링
       ※ 이미지 수신 없음. 수치 데이터만 수신.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  PHASE B: 착륙 후 — 물리적 SD카드 회수
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  캔위성 회수 → SD카드 분리 → RTX 3080 데스크톱에 직접 삽입
  ※ 네트워크 전송 없음. Wi-Fi / LAN / 서버 없음.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  PHASE C: 분석 — RTX 3080 로컬 처리
```
